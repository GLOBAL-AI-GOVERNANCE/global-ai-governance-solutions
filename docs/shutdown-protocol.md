# Shutdown Protocol

No AI system should be deployed unless it can be stopped.

## Shutdown Triggers

Shutdown review begins when an AI system:

- Causes harm
- Produces materially false outputs
- Exceeds approved scope
- Leaks sensitive data
- Demonstrates unsafe autonomy
- Loses required monitoring
- Violates law or policy
- Fails required performance thresholds
- Cannot be audited
- Cannot be controlled by assigned owners

## Shutdown Decision Options

- Temporary pause
- Partial restriction
- User access removal
- Vendor access suspension
- Rollback to prior version
- Full shutdown
- Permanent retirement

## Required Shutdown Documentation

- System name
- Owner
- Risk tier
- Trigger event
- Impact assessment
- Decision authority
- Shutdown method
- Communication plan
- Remediation plan
- Restart conditions

## Restart Rule

A stopped AI system may not restart until evidence shows that the failure condition has been corrected and required controls are operating.
