# Approval Gates

AI deployment approval must be based on evidence, not optimism.

## Gate 1: Inventory Complete

Required:

- System name
- Use case
- Owner
- Vendor or internal source
- Data used
- User group
- Impact area
- Risk tier

## Gate 2: Ownership Confirmed

Required:

- Business owner
- Technical owner
- Security owner
- Privacy/data owner
- Legal/compliance reviewer
- Executive owner if high impact

## Gate 3: Risk Tier Approved

Required:

- Risk tier
- Rationale
- Impact assessment
- Autonomy assessment
- Data sensitivity assessment
- Failure mode assessment

## Gate 4: Evidence Reviewed

Required:

- Test results
- Known limitations
- Red team or adversarial testing for high-risk systems
- Security review
- Privacy review
- Legal review when applicable
- Monitoring plan
- Shutdown plan

## Gate 5: Controls Verified

Required:

- Human authority
- Least privilege
- Logging
- Monitoring
- Incident response
- Appeal/correction process when public impact exists
- Shutdown path

## Gate 6: Deployment Decision

Decision options:

- Approve
- Approve with restrictions
- Require more evidence
- Pilot only
- Reject
- Suspend
