# Monitoring and Audit

AI governance does not end at deployment.

## Monitoring Requirements

Every deployed AI system should have monitoring appropriate to its risk tier.

Monitor for:

- Drift
- Failure
- Bias
- Hallucination
- Misuse
- Security exposure
- Data leakage
- Scope expansion
- Vendor changes
- User complaints
- Appeal outcomes
- Incident trends

## Audit Trail Requirements

Audit trails should capture:

- System identity
- Version
- Inputs where appropriate and lawful
- Outputs where appropriate and lawful
- User actions
- Agent actions
- Approval decisions
- Override decisions
- Incident events
- Shutdown events

## Review Cadence

Suggested cadence:

- Low: annual review
- Moderate: semiannual review
- High: quarterly review
- Critical: monthly or quarterly review
- Frontier: continuous monitoring with formal recurring review
