# Vendor Accountability

Vendor AI requires governance before purchase, before deployment, and after deployment.

## Core Rule

No vendor AI without contractual accountability.

## Minimum Vendor Questions

- What data is processed?
- Is data retained?
- Is data used for training?
- Where is data stored?
- Who are the subprocessors?
- What security controls exist?
- What audit logs are available?
- How are incidents reported?
- How are model updates disclosed?
- Can data be deleted or returned?
- What happens at termination?

## Contract Requirements

Contracts should include:

- Data protection obligations
- Security obligations
- Incident notification timelines
- Audit rights
- Subprocessor disclosure
- Model update notice
- Data deletion and return
- Use restrictions
- Liability allocation
- Termination rights

## Buyer Responsibility

The buyer remains accountable for deployment, monitoring, correction, and shutdown.
