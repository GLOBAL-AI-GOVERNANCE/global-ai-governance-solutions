# AI Deployment Approval Checklist

Use this checklist before approving an AI system for pilot, production, or expansion.

## Gate 1: Discovery

- [ ] System name documented
- [ ] Use case documented
- [ ] Vendor/internal source documented
- [ ] Data used documented
- [ ] Users affected documented
- [ ] Business or mission process documented
- [ ] Shadow AI status checked

## Gate 2: Ownership

- [ ] Business owner assigned
- [ ] Technical owner assigned
- [ ] Security owner assigned
- [ ] Privacy/data owner assigned where needed
- [ ] Legal/compliance reviewer assigned where needed
- [ ] Executive owner assigned for High, Critical, or Frontier systems

## Gate 3: Risk

- [ ] Risk tier assigned
- [ ] Risk tier rationale documented
- [ ] Autonomy level assessed
- [ ] Data sensitivity assessed
- [ ] Public impact assessed
- [ ] Cyber exposure assessed
- [ ] Shutdown feasibility assessed

## Gate 4: Evidence

- [ ] Test results attached
- [ ] Limitations documented
- [ ] Security review complete
- [ ] Privacy review complete where needed
- [ ] Legal review complete where needed
- [ ] Red team review complete for High+ systems
- [ ] Vendor documentation reviewed if applicable

## Gate 5: Controls

- [ ] Human authority preserved
- [ ] Least privilege enforced
- [ ] Audit trail active
- [ ] Monitoring active
- [ ] Incident response path active
- [ ] Shutdown path tested
- [ ] Appeal/correction process active for public-impact systems

## Decision

- [ ] Approved
- [ ] Approved with restrictions
- [ ] Pilot only
- [ ] More evidence required
- [ ] Rejected
