# Procurement AI Review Gate

## Core Rule

Any procurement involving AI must pass AI governance review before purchase, renewal, or expansion.

## Trigger Events

Review is required when:

- A product includes AI capability
- A vendor adds AI capability to an existing product
- A model processes enterprise data
- A system automates decisions
- A tool uses autonomous agents
- A vendor changes model behavior or data handling
- A contract is renewed for AI-enabled services

## Minimum Review

- AI use case
- Data handling
- Risk tier
- Owner assignment
- Security review
- Privacy review
- Legal review
- Vendor accountability
- Monitoring plan
- Shutdown or termination path

## Procurement Rule

No AI vendor should bypass ownership, evidence, authority, or control.
