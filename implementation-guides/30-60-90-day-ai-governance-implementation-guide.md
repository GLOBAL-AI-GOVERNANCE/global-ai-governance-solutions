# 30/60/90-Day AI Governance Implementation Guide

## Days 1–30: Discover And Own

Goal: make AI visible.

Actions:

- Create AI governance owner group
- Publish acceptable AI use baseline
- Launch AI system inventory
- Identify existing AI vendors
- Identify shadow AI use
- Assign business and technical owners
- Start risk-tier classification
- Freeze high-risk unowned deployments

Deliverables:

- Initial AI inventory
- Owner map
- Vendor list
- Shadow AI register
- Draft governance cadence

## Days 31–60: Classify And Control

Goal: match controls to risk.

Actions:

- Complete risk tier review
- Apply approval gates
- Review sensitive data use
- Review autonomous agent permissions
- Review vendor AI contracts
- Establish monitoring and incident response
- Define shutdown paths for High+ systems

Deliverables:

- Risk tier register
- Vendor review scorecards
- Agent permission matrix
- Incident response playbook
- Shutdown protocol

## Days 61–90: Govern And Improve

Goal: make governance operational.

Actions:

- Launch executive AI governance board
- Approve, restrict, or retire systems
- Establish recurring monitoring cadence
- Run tabletop incident exercise
- Publish internal AI deployment approval process
- Set quarterly review cycle

Deliverables:

- Board charter
- Approved AI register
- Restricted AI register
- AI incident exercise report
- v1 governance metrics dashboard
