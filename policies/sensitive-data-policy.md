# Sensitive Data Policy

## Core Rule

No sensitive data without protection.

## Sensitive Data Categories

Sensitive data includes:

- Personal information
- Health information
- Financial information
- Biometric information
- Legal information
- Security information
- Proprietary information
- Regulated data
- Classified or controlled information
- Mission-critical operational data

## Required Controls

Before sensitive data is used with AI:

- Data owner approval is required
- Purpose must be documented
- Minimum necessary data must be used
- Access must be restricted
- Retention must be defined
- Logging must be enabled
- Vendor terms must be reviewed
- Data deletion or return rights must exist
- Security and privacy reviews must be completed

## Prohibited Use

Sensitive data may not be entered into unapproved AI tools.
