# Vendor AI Policy

## Core Rule

No vendor AI without contractual accountability.

## Required Vendor Controls

Vendor AI contracts should address:

- Data use
- Data retention
- Data deletion
- Security controls
- Privacy controls
- Subprocessors
- Audit rights
- Incident notification
- Model updates
- Service-level expectations
- Indemnity and liability
- Termination rights

## Buyer Accountability

Using a vendor does not transfer responsibility away from the organization.

Accountability follows both:

- The vendor that provides the AI system
- The organization that deploys it

## Vendor Change Rule

Material vendor changes require renewed review when they affect:

- Model behavior
- Data handling
- Security posture
- Subprocessors
- Approved use
- Monitoring
- Shutdown path
